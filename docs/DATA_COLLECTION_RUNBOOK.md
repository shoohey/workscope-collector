# 2週間後のデータ回収ランブック

> パイロット薬局の WorkScope Collector で 2 週間蓄積したデータを USB で回収し、
> 解析（Phase 2）に進めるための具体手順。
> セキュリティ要件（個情法・薬剤師法）と業務継続を両立する設計。

---

## 0. 回収日決定 〜 訪問前（1日前まで）

| | 項目 |
|---|---|
| ☐ | 薬局担当者と回収日時を確定（業務繁忙時間帯は避ける） |
| ☐ | **暗号化 USB**（BitLocker To Go / Kingston IronKey 等）を新品で準備 |
| ☐ | USB に空き容量 8GB 以上あること（30 日分のスクショで最大 5GB を想定） |
| ☐ | 同意書の控え（自社保管分）と、回収記録用紙（後述「回収記録テンプレ」）を持参 |
| ☐ | データ回収後の運用方針（**継続録画 / 一時停止 / アンインストール** のいずれか）を事前にすり合わせ |
| ☐ | 訪問者は最低 2 名推奨（4 つ目セクション参照） |

## 1. 訪問・挨拶（5 分）

| | 項目 |
|---|---|
| ☐ | 薬局責任者・店長に挨拶し、回収手順を口頭で再確認 |
| ☐ | 「これから USB に既存データをコピーして持ち帰ります」と明示 |
| ☐ | 質問・不安があればその場で受け止め、回収を中止する選択肢も提示 |

## 2. 録画停止（重要・最初に実施）

データを動的に書き換えている最中にコピーすると、JSONL の最終行が破損する可能性がある。
**まずは録画を確実に止める** こと。

| | 項目 |
|---|---|
| ☐ | タスクトレイのアイコンを右クリック → 「**一時停止**」を選択 |
| ☐ | アイコンが **灰色丸**（paused）に切り替わったことを目視確認 |
| ☐ | アイコン右クリック → 「ステータス」で `一時停止中` と表示されること |
| ☐ | 念のため 5 秒待つ（書き込み中のファイルが flush される時間を取る） |

---

## 3. データのコピー（10〜20 分）

PII はマスク済みだが、**USB 持ち出しは個人情報持ち出しと同等の管理** を行う。

### 3-1. コピー対象
```
%APPDATA%\WorkScope\data\
├── screenshots\   ← マスク済みJPEG（重要）
└── events\        ← JSONL イベントログ（重要）

%APPDATA%\WorkScope\logs\
├── main.log
├── collector.log
└── install.log    ← トラブルシュート用、PIIなし
```

### 3-2. 推奨コピー手順
**robocopy（標準コマンド）で安全にコピー**：

```cmd
:: 暗号化USBが E: ドライブ、薬局名 sample-pharmacy、回収日 2026-05-18 と仮定
set DEST=E:\workscope-collected\sample-pharmacy\2026-05-18
mkdir "%DEST%"
robocopy "%APPDATA%\WorkScope\data\screenshots" "%DEST%\screenshots" *.jpg /E /R:3 /W:5 /MT:4 /NP /TEE /LOG+:"%DEST%\copy.log"
robocopy "%APPDATA%\WorkScope\data\events" "%DEST%\events" *.jsonl /E /R:3 /W:5 /MT:4 /NP /TEE /LOG+:"%DEST%\copy.log"
robocopy "%APPDATA%\WorkScope\logs" "%DEST%\logs" *.log /E /R:3 /W:5 /MT:4 /NP /TEE /LOG+:"%DEST%\copy.log"
```

| | 項目 |
|---|---|
| ☐ | コピー完了後、`copy.log` に `Files :    Copied : N` と "Failed : 0" を確認 |
| ☐ | `events\` 内の JSONL が **薬局のローカル件数と一致** している（薬局担当者と一緒に確認） |
| ☐ | スクショ枚数を `dir /B "%DEST%\screenshots" | find /C /V ""` で確認 |

### 3-3. コピー検証
| | 項目 |
|---|---|
| ☐ | 最新の `events\YYYY-MM-DD.jsonl` をテキストエディタで開き、最終行が完全な JSON になっている（途切れていない） |
| ☐ | スクショ 3 枚をランダムサンプリングしてブラウザで開く → **PII が黒塗り済み**であることを目視確認（薬剤師さん立ち会い） |

---

## 4. 整合性チェック（5 分）

| | 項目 |
|---|---|
| ☐ | コピー先の SHA256 を取って、後の社内検証用にメモ：<br>`certutil -hashfile "%DEST%\events\2026-05-18.jsonl" SHA256` |
| ☐ | コピー先と元データのファイル数とバイトサイズを比較<br>`dir "%APPDATA%\WorkScope\data\screenshots" /B \| find /C /V ""` 等 |
| ☐ | 失敗 / エラーがあった場合はコピーを再実行（最大 3 回） |

---

## 5. 元データの取り扱い（薬局側との合意で 1 つを選択）

回収後、**薬局側のローカルデータをどうするか**は事前に決めて合意しておく。

### 5-A. 継続録画（解析と並行運用）
| | 項目 |
|---|---|
| ☐ | アイコン右クリック → 「再開」 → 緑に戻る |
| ☐ | 元データはそのまま残す（30/90日 自動削除に任せる） |
| ☐ | 翌週以降の追加回収日程を取り決め |

### 5-B. 一時停止（録画は止めて、改善後再開）
| | 項目 |
|---|---|
| ☐ | アイコンは灰色丸のまま放置 |
| ☐ | 元データはそのまま残す |
| ☐ | 改善版 EXE を配布する目処を伝える |

### 5-C. アンインストール（パイロット完了）
| | 項目 |
|---|---|
| ☐ | USB の `uninstall.bat` を右クリック → 管理者として実行 |
| ☐ | 「収集データも削除しますか？」 → **Y を選択（完全削除）** |
| ☐ | `%APPDATA%\WorkScope\` フォルダが消えていることを確認 |
| ☐ | スタートアップ登録解除を確認：`reg query "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v WorkScope` → "見つかりません" と返る |

## 7. 帰社後の処理（社内）

| | 項目 |
|---|---|
| ☐ | USB を **金庫 or 鍵付きキャビネット** に保管 |
| ☐ | データを社内サーバの **暗号化された保管領域**にコピー（仮: `secured/workscope/<pharmacy>/<date>/`） |
| ☐ | コピー後、SHA256 を再計算して訪問時のメモと一致を確認 |
| ☐ | 暗号化 USB は **データ移送専用** とし、解析作業時は別領域で扱う |
| ☐ | 訪問記録テンプレを社内ナレッジに保管（紙原本は鍵付き保管） |

## 8. 解析フェーズへの引き継ぎ（Phase 2）

| | 項目 |
|---|---|
| ☐ | `analyzer/ROADMAP.md` の Phase 2 に従って `session_segmenter.py` 等の実装着手 |
| ☐ | 解析環境はオフライン PC または閉じた仮想環境（クラウドへの送信を避ける） |
| ☐ | 解析結果（業務マップ・自動化候補）を成果物として薬局へ提示 |
| ☐ | 提示時には **マスク済み画像 / 匿名化済みデータ** のみを使用、原データは社内に留める |

## 9. インシデント発生時の連絡先 / 対応

```
USB 紛失・盗難の可能性
   ├── 即時: 薬局担当者・社内責任者へ電話連絡
   ├── BitLocker To Go なら遠隔ワイプは不可だが、暗号化済みなので解読困難
   ├── ログを保全（USB 配布記録 / コピー記録 / 紛失時刻）
   └── 個人情報保護委員会への報告要否を法務と確認

USB 内データに PII 残留が発覚
   ├── 即時: 該当ファイルを削除（社内サーバ + USB）
   ├── 削除前にハッシュとファイル数を保全
   ├── 該当パターンを masker.py に追加してリリース
   └── パイロット薬局に「再録画＋整合性確認」の協力を依頼
```
