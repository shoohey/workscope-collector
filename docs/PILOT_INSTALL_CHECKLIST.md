# パイロット薬局 導入チェックリスト

> 明日のパイロット店舗訪問で実施する手順書。
> このリストを順番に上から下へ進めれば、初日インストールから動作確認まで完了する。
> 各項目の **チェック欄** にレ点を入れながら進める。

---

## 0. 訪問前の準備（前日夜 / 当日朝）

| | 項目 |
|---|---|
| ☐ | USB（暗号化推奨。BitLocker To Go / Kingston IronKey 等） |
| ☐ | USB に最新の `WorkScope.exe / install.bat / uninstall.bat / README_install.txt / consent_form.html / operation_guide.html / data_handling_policy.html` を入れる |
| ☐ | `consent_form.html` を A4 で 2 枚以上印刷（薬剤師さん全員ぶん＋控え） |
| ☐ | 同意書の事業者欄（株式会社TRIBE / 担当 / 電話 / メール）が記入済み |
| ☐ | レセコン端末の OS / メーカー / 製品名（カルテにメモ済み）|
| ☐ | 薬局担当者（村上さん / 現場担当者）と訪問時刻の最終確認 |
| ☐ | 万一持ち帰る必要が出た場合の **電源 / ネット遮断時のバックアップ手順** を頭で押さえる |

| ☐ | 業務の繁忙時間を避けてもらえるか確認（昼前 / 夕方の処方ピーク前後を避ける） |

## 2. 業務説明・同意取得（10〜15 分）

| | 項目 |
|---|---|
| ☐ | `data_handling_policy.html` を画面で見せながら **オンデバイス完結** / **生スクショは保存しない** / **マスクは strict mode** を口頭説明 |
| ☐ | 「タスクトレイに緑丸が出ている＝録画中、グレー丸＝一時停止」を見せる |
| ☐ | 一時停止 / アンインストール / データ削除はいつでもできる旨を説明 |
| ☐ | `consent_form.html` を印刷したものに薬剤師さん全員の署名を取得 |
| ☐ | 控え 1 枚を薬局に渡す（保管推奨） |
| ☐ | 質問が出たらその場で回答。少しでも納得が得られなければインストールしない |

## 3. インストール（10 分）

| | 項目 |
|---|---|
| ☐ | レセコン PC を借りる前に **業務中に開いていた画面のスクショを念のため作業前にユーザー側で取ってもらう** |
| ☐ | USB を PC に差し込む |
| ☐ | `install.bat` を右クリック → **管理者として実行** |
| ☐ | 黒いコンソールに `[5/5] インストールが完了しました。` と出る |
| ☐ | コンソールにエラー（赤字 / "失敗" / "エラー"）が無いことを目視確認 |
| ☐ | `%APPDATA%\WorkScope\logs\install.log` を一度開いて「インストール完了」が末尾にあるか確認 |
| ☐ | タスクトレイに緑色のアイコンが表示されている |
| ☐ | アイコンを右クリック → 「録画中（イベント数: 0 / セッション時間: 00:00:0X）」と表示される |

---

## 4. 動作確認（10 分）

実画面でマスキングが正しく動くかを薬剤師さん立ち会いのもとで確認する。

### 4-1. 基本フロー
| | 項目 |
|---|---|
| ☐ | レセコンを 1 〜 2 件操作してもらう（実患者の操作で OK。マスクされるので問題なし） |
| ☐ | アイコン右クリック → 「直近のスクショを確認」 → 一覧が出る |
| ☐ | 直近 1 件をクリックし、ブラウザ／ペイントで開く |
| ☐ | **患者氏名・電話・住所・保険者番号・マイナンバー・生年月日が黒塗りされている**ことを薬剤師さんと一緒に目視確認 |
| ☐ | 黒塗りされていない PII を発見した場合は **その場で一時停止** → 連絡先（私）にすぐ電話／LINE |

### 4-2. 一時停止 / 再開
| | 項目 |
|---|---|
| ☐ | アイコン右クリック → 「一時停止」 → アイコンが灰色に切り替わる |
| ☐ | 数秒後、アイコン右クリック → 「再開」 → 緑に戻る |
| ☐ | この操作後にレセコンを操作 → 直近スクショが新しく増えていることを確認 |

### 4-3. ブロックリスト動作（任意）
| | 項目 |
|---|---|
| ☐ | パスワード管理アプリ（1Password 等）を開く → スクショが取られないことを確認（薬局に該当アプリがある場合のみ） |

## 5. 帰り際チェック（5 分）

| | 項目 |
|---|---|
| ☐ | `%APPDATA%\WorkScope\data\events\YYYY-MM-DD.jsonl` が 1 ファイル以上生成されている |
| ☐ | `%APPDATA%\WorkScope\data\screenshots\` に JPEG が 5 枚以上ある |
| ☐ | `%APPDATA%\WorkScope\logs\collector.log` に ERROR が出ていない（一時的な WARN は OK） |
| ☐ | 「2 週間後の回収日」を薬局担当者と再確認（カレンダー登録） |
| ☐ | 連絡先（電話 / LINE / メール）を交換し、何かあればすぐ連絡をもらう |

## トラブル時の即時対応フロー

```
PII 漏れを発見
   │
   ├── 即時: タスクトレイの「一時停止」をクリック
   ├── 担当者（私）に電話 / LINE で報告
   ├── %APPDATA%\WorkScope\data\screenshots\ の漏洩疑いファイルを退避（ZIP 化して暗号化 USB に）
   └── アンインストール（uninstall.bat → Y で完全削除）するか、修正版 EXE 配布まで一時停止のまま運用
```
